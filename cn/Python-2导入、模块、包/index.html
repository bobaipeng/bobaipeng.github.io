1. 导入的目的是什么？

将Python的一些功能函数导入到当前的脚本中使用，很多函数不能导入是不能使用的，除了内置函数如print()

2. 导入的位置一般在什么地方？为什么？

导入的位置一般在头注释下面，因为程序是自顶向下进行执行的，头注释之后就可以进行导入了，然后进行函数调用

3. 模块与包

3.1 在导入的时候导入的就是模块与包，那么我们该如何理解模块和包呢

模块： 任何 的 *.py 文件都可以当作模块 用 import 导入,即每个*.py 文件都可看作是一个模块

包： 包含一个__init__.py文件和其他模块文件、以及子目录及文件，

$\textcolor{red}{注意:} $ ==其中__init__.py可以没有，模块文件里的属性、函数、类只有用模块才能调用出来，所以导入模块一般情况下是必须的==

3.3 导入模块

导入模块的时候使用import 和from…import…

from ...import *可以导入包下所有模块，但是占用系统损耗太大，一般不用，只精确导入自己需要的模块等

5. __init__.py的作用详解

  • 在python3.3之前，此文件必须在包内存在，这个包才是module package ，在3.3之后这个文件不是必须的了，可以不存在
  • 这个文件没有强制要求，可以是一个空文件，也可以有内容
  • 包的作用是组织多个模块或者子包提供功能集，而导入包就相当于导入这个包的__init__.py文件，因为实际使用的时候这个文件的具体作用是导入该包内所包含的模块，即在此文件中使用import... 、from....import...
  • 具体的过程是在导入含__init__.py文件的包时会首先执行该文件内的代码，而此文件内有导入代码，就会将报下模块、子包变成包内成员，这样以后使用的时候只需要导入包就行了，不需要关心包内的模块（当然字样会有自动导入，增加系统开销，也可让调用着自己精确导入所需的模块）

==具体使用过程中使用__init__.py自动加载包的优点：==使用它自动加载包，我们就可以把包下面的模块、模块成员、子包、子包模块等都当成该包的成员，我们在使用时只需要导入该包，然后通过包名.的方式进行调用
